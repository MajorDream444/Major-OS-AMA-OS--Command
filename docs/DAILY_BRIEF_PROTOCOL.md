# Daily Brief Protocol

## Purpose

The Daily Corporate Brief turns Major AI OS into a 30-day aggressive execution cycle.

Operating mode: hybrid aggressive.

Primary objective: every day must produce:

- one revenue action
- one content action
- one logged system improvement

## Source Of Truth

```text
GitHub = truth
Airtable = business state
Notion = human dashboard
Obsidian / Graphify = memory graph
Substack = expression and content engine
n8n = automation engine
OpenClaw = agent runtime
```

No implementation should rely on chat memory as the operating record.

## Execution Loop

```text
Plan -> Build -> Run -> Log -> Improve
```

## Daily Brief Sections

### 1. Revenue Action

Records the single action most likely to create money, sales motion, lead conversion, offer clarity, or client movement today.

Required fields:

- status
- owner agent
- target
- next action
- evidence link or local handoff path

### 2. Content Action

Records the single expression/output action most likely to build authority today.

Required fields:

- status
- owner agent
- platform or channel
- asset type
- next action
- evidence link or local handoff path

### 3. System Improvement

Records the one improvement that makes the operating system more reusable, visible, automated, or reliable.

Required fields:

- status
- owner agent
- system area
- change made
- next action
- evidence link or local handoff path

### 4. Blocker

Records the highest-friction issue that needs Major, Codex, OpenClaw, or future automation attention.

Required fields:

- status
- owner agent
- blocker
- needed decision
- next action
- evidence link or local handoff path

## Local-Only Rule For Current Build

The Command Center may display mock daily brief data locally.

Do not connect Airtable, Notion, Gmail, HeyGen, Remotion, Telegram, GitHub APIs, or n8n yet.

## Handoff Rule

Every brief cycle must leave a handoff in GitHub.

Minimum handoff:

- date
- completed revenue action
- completed content action
- completed system improvement
- unresolved blocker
- next recommended task
